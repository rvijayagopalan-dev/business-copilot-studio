# Gaps & Checklist — API Copilot

Use this checklist to track missing capabilities, risks, and next steps for the API Copilot product.

Checklist

- [ ] CI/Automation: integrate tests and linting into CI for generated artifacts.
- [ ] Policy plugins: add organization-specific policy enforcement (Spectral rules, security policies).
- [ ] PII handling: ensure generated sample data does not contain real PII; add privacy checks.
- [ ] Accessibility: ensure generated documentation and demos follow accessibility practices.
- [ ] Metrics & Observability: add automated recommendations for metrics and SLOs.
- [ ] Deployment pipelines: templates for publishing mock servers and SDKs.
- [ ] Multi-language SDK generation coverage (TypeScript, Java, Python, C#).
- [ ] Template prompts registry: centralize proven prompts and version them.
- [ ] Evaluate hallucination risk: add verification steps for generated specs (linting, example validation).
- [ ] Feedback loop: collect user feedback and automatically tune prompts or rules.

Use this file to log actions, owners, and completion dates as the project matures.
