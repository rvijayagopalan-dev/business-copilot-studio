Use cases, sample input queries, representative outputs, and the agents typically invoked by the platform.

Use Case: Generate OpenAPI Spec from Intent

Input Query
"Design a Customer API for Banking: endpoints to create, get, update, delete customers. Return OpenAPI 3.1 with request/response schemas and examples."
Output Response (representative)
{ "openapi": "3.1.0", "info": {"title":"Customer API"}, "paths": { "/customers": {"post": {...}}, "/customers/{id}": {"get": {...}} }, "components": {"schemas": {...} }}
Agents Invoked
Architect Agent Developer Agent Documentation Agent
Notes
Follow-up prompts may ask for security schemes (OAuth2), pagination, and error models.

Notes

Showcase Guidance

  • Each prompt should include context: target API, security constraints, and desired artifact format (OpenAPI, Postman, SDK language).
  • Start with small, deterministic requests and iterate—ask for tests and examples in the same prompt.
  • Record which agents were invoked in the prompt metadata to replicate the workflow.

Files: `SKILL.md`, `prompt.md`, `claude-prompts.md` should include the canonical prompt examples used for the above use cases.
