# Copilot Template Skill

## Purpose

This template defines the standard structure for Claude-based copilot products in this workspace.

## Core Structure

A generic copilot product should include:

- product knowledge and capability definitions
- behavior rules and interaction guidance
- prompt examples and templates
- onboarding documentation

## Recommended Files

- `SKILL.md` — product capabilities, roles, outputs, and knowledge definitions.
- `copilot-instructions.md` — behavior rules, output style, and interaction guidance.
- `prompt.md` — example prompts, templates, and usage patterns.
- `README.md` — purpose, folder structure, and onboarding notes.

## How to Use

1. Duplicate this folder for a new copilot product.
2. Replace generic content with product-specific capabilities.
3. Keep the file set consistent across copilots.
4. Update examples and instruction rules for the new domain.

## Template Guidance

- Keep `SKILL.md` focused on what the product knows and can do.
- Keep `copilot-instructions.md` focused on how the copilot should behave.
- Keep `prompt.md` focused on user-facing examples and templates.
- Keep `README.md` focused on folder-level intent and onboarding.
